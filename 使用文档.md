# 📷 二维码 H5 照片画廊 — 使用文档

## 一、项目概述

手机扫码 → 自动定位到对应人物照片 → 上下滑动浏览其他人。

- **适用场景**：团队介绍、活动展示、产品画册、婚礼相册等
- **核心特性**：数据驱动、扫码定位、单列大图流、秒开、全平台适配
- **技术栈**：纯静态 HTML，零框架依赖，永久免费部署

---

## 二、文件结构

```
项目文件夹/
├── index.html          ← ⭐ 主画廊页（扫码后打开的页面）
├── qr-codes.html       ← 二维码生成页（本地 Canvas 生成，无外部依赖）
├── data.js             ← ⭐ 人物数据配置（你主要编辑这个文件）
├── photos/             ← 放照片的文件夹
├── generate-photos.html← 占位图生成工具
└── 使用文档.md          ← 本文档
```

## 三、快速上手（改内容只需 2 步）

### 第 1 步：编辑数据

打开 `data.js`，修改 `PEOPLE_DATA` 数组：

```js
const PEOPLE_DATA = [
  {
    id: 1,                              // 编号（对应二维码 ?person=1）
    name: "小明",                        // 姓名
    photo: "photos/xiaoming.jpg",       // 照片路径（为空则自动生成占位图）
    description: "喜欢摄影和旅行",        // 简介
    tags: ["摄影", "旅行", "咖啡"]        // 标签
  },
  // ... 复制粘贴来加更多人
];
```

**⚠️ 重要**：每次改完数据后，把上面一行的版本号 +1：

```js
window.__DATA_VERSION__ = 7;  // ← 改这个数字，比如 7 → 8
```

然后打开 `index.html`，找到开头的这一行也改成一样：

```js
window.__BUILD__ = 7;  // ← 保持一致
```

> 💡 这个版本号确保所有人的手机都会强制加载最新内容，不会有缓存残留。

### 第 2 步：推送更新

```bash
git add . && git commit -m "更新内容" && git push
git push gitee main    # 同时推到 Gitee
```

**然后**去 Gitee 网页 → 仓库 → 服务 → Gitee Pages → 点击「更新」按钮。

等 1 分钟，扫码即看到最新内容。

## 四、照片处理

有 3 种方式：

| 方式 | 操作 |
|------|------|
| **A. 本地文件** | 把照片放 `photos/` 文件夹，`photo` 填 `"photos/xxx.jpg"` |
| **B. 网络图片** | `photo` 直接填图片网址，如 `"https://example.com/pic.jpg"` |
| **C. 不填** | `photo` 留空 `""`，自动生成彩色 Canvas 占位图 |

## 五、增删人物

### ➕ 加人

在 `data.js` 的 `PEOPLE_DATA` 数组里复制粘贴一段：

```js
{id: 4, name: "新人物", photo: "", description: "", tags: []},
```

⚠️ `id` 必须唯一，对应二维码的 `?person=ID`

### ➖ 删人

直接删掉对应那段 `{...}` 即可。

### 📊 从 3 人扩展到 30 人

粘贴 27 段配置，每个改 `id`（1~30），有照片就填 `photo` 路径。

## 六、二维码

### 生成二维码

打开 `qr-codes.html`：
1. 顶部自动填入 **Gitee Pages 地址**（国内推荐）
2. 也可以点「GitHub Pages」切到海外备用地址
3. 二维码纯前端 Canvas 生成，**不依赖任何外部 API**
4. 点「⬇ 下载二维码」保存 PNG 图片
5. 浏览器按 `Ctrl+P` 打印，每码一页 A4

### 二维码网址格式

```
你的网址/?person=编号
```

例如：
- `https://jiapeng-xiao.gitee.io/photo-gallery/?person=1` → 扫码定位到第1人 🇨🇳
- `https://xjp0108.github.io/photo-gallery/?person=2` → 扫码定位到第2人 🌍

## 八、微信扫码说明

微信扫外链二维码会弹出安全提示："非微信官方网页，请确认是否继续访问"

**这是微信对所有非白名单域名的统一行为，不是网页问题。**

处理方法：
1. 点「继续访问」即可正常浏览
2. 或用手机相机/浏览器扫码，不经过微信，直接打开

如需去掉此提示，需：自有域名 + ICP 备案 + 企业资质 + 微信白名单申请（耗时 1 个月+，花费数千元）。

## 九、常见问题

### Q：修改后扫码不更新？

A：本项目已做三重防缓存机制：
1. HTML meta 标签禁止缓存
2. data.js 带版本号参数加载（`data.js?v=7`）
3. 版本号校验（不一致自动强制刷新）

如果仍不更新，微信内点右上角 `···` → 刷新，或手动下拉刷新。

### Q：二维码扫不出来？

A：分两种情况：
1. **二维码本身扫不出来**：检查 `qr-codes.html` 是否正确渲染了二维码（应该是 Canvas 绘制，不是空白）
2. **扫出来打不开网页**：确认 Gitee Pages 已启动。如果 Gitee 抽风，切 GitHub Pages 备用

### Q：照片不显示？

A：检查三点：
1. 照片文件是否在 `photos/` 文件夹内
2. `photo` 路径是否写对（区分大小写）
3. 是否已推送到线上

### Q：想改配色/样式？

A：编辑 `index.html` 中 `<style>` 标签里的 CSS。主体色是 `#0f0f1a`（深色背景）、`#ff6b6b`（高亮红色）。

## 十、技术要点

| 项 | 说明 |
|------|------|
| 布局 | 单列大图流，4:5 图片比例，类小红书风格 |
| 扫码定位 | URL 参数 `?person=N` + 平滑滚动 + 高亮动画 |
| 降级机制 | 照片加载失败自动切换到 Canvas 占位图 |
| 移动端 | iOS Safari / Android Chrome / 微信内置浏览器全适配 |
| 安全区域 | iPhone 刘海屏 + 底部横条自动避让 |
| QR 生成 | 纯前端 Canvas（qrcodejs），不依赖任何外部 API |
| 缓存控制 | 三重机制：meta no-cache + 版本号 URL + 版本校验 |
| 超时处理 | 8 秒超时自动提示 + 重试 + 刷新按钮 |
