# 📷 二维码 H5 照片画廊 — 使用文档

## 一、项目概述

手机扫码 → 自动定位到对应人物照片 → 上下滑动浏览其他人。

- **适用场景**：团队介绍、活动展示、产品画册、婚礼相册等
- **核心特性**：数据驱动、扫码定位、单列大图流、秒开、全平台适配
- **技术栈**：纯静态 HTML，零框架依赖，永久免费部署

---


## 三、快速上手（3 步）

### 第 1 步：改数据

打开 `index.html`，找到 `<script>` 里的 `PEOPLE_DATA`，按需修改：

```js
const PEOPLE_DATA = [
  {
    id: 1,                              // 编号（二维码 ?person=1）
    name: "小明",                        // 姓名
    photo: "",                          // 照片路径（为空则自动生成占位图）
    description: "喜欢摄影和旅行",        // 简介
    tags: ["摄影", "旅行", "咖啡"]        // 标签
  },
  {
    id: 2,
    name: "小红",
    photo: "",
    description: "热爱画画的设计师",
    tags: ["设计", "绘画", "音乐"]
  },
  // ... 复制上段继续加人
];
```

### 第 2 步：放照片（可选）

有 3 种方式：

| 方式 | 操作 |
|------|------|
| **A. 本地文件** | 把照片放 `photos/` 文件夹，`photo` 填 `"photos/xxx.jpg"` |
| **B. 网络图片** | `photo` 直接填图片网址，如 `"https://example.com/pic.jpg"` |
| **C. 不填** | `photo` 留空 `""`，自动生成彩色占位图 |

### 第 3 步：推送更新

```bash
git add . && git commit -m "更新内容" && git push
```

等 1 分钟 GitHub Pages 自动部署，扫码即刷新（无需清缓存）。

## 四、增删人物

### ➕ 加人

在 `PEOPLE_DATA` 数组里复制粘贴一段，改 `id`、`name`：

```js
{id: 4, name: "新人物", photo: "", description: "", tags: []},
```

⚠️ `id` 必须唯一，对应二维码的 `?person=ID`

### ➖ 删人

直接删掉对应那段 `{...}` 即可。

### 📊 从 3 人扩展到 30 人

粘贴 27 段配置，每个改 `id`（1~30），有照片就填 `photo` 路径。

## 五、二维码

### 生成二维码

打开 `qr-codes.html`，顶部输入部署网址，二维码自动生成。

### 二维码网址格式

```
你的网址/?person=编号
```

例如：
- `https://xjp0108.github.io/photo-gallery/?person=1` → 扫码定位到第1人
- `https://xjp0108.github.io/photo-gallery/?person=2` → 扫码定位到第2人

### 打印二维码

`qr-codes.html` 自带打印样式，浏览器按 `Ctrl+P` 打印，每码一页 A4。

## 六、部署方式

### 方案 A：GitHub Pages（推荐，已部署）

| 步骤 | 操作 |
|------|------|
| 上传文件 | git push 或网页拖拽上传到 GitHub 仓库 |
| 开启 Pages | Settings → Pages → Branch 选 main → Save |
| 访问地址 | `https://用户名.github.io/仓库名/` |

✅ 永久免费 | ✅ HTTPS | ❌ 国内偶尔稍慢

### 方案 B：Gitee Pages（国内更快）

| 步骤 | 操作 |
|------|------|
| 上传文件 | git push 或网页上传到 Gitee 仓库 |
| 实名认证 | 设置 → 实名认证（需身份证，通常 1 工作日审核） |
| 开启 Pages | 仓库页 → 服务 → Gitee Pages → 启动 |
| 访问地址 | `https://用户名.gitee.io/仓库名/` |

✅ 国内秒开 | ✅ 永久免费 | ⚠️ 需实名

## 七、微信扫码说明

微信扫外链二维码会弹出安全提示："非微信官方网页，请确认是否继续访问"

**这是微信对所有非白名单域名的统一行为，不是网页问题。**

处理方法（告诉客户）：
1. 点「继续访问」即可正常浏览
2. 或用手机相机/浏览器扫码，不经过微信，直接打开

如需去掉此提示，需：自有域名 + ICP 备案 + 企业资质 + 微信白名单申请（耗时 1 个月+，花费数千元）。

## 八、常见问题

### Q：修改后扫码不更新？

A：已内置禁用缓存机制，修改后推送代码等 1 分钟即可生效。如果仍不更新，微信内点右上角 `···` → 刷新。

### Q：照片不显示？

A：检查三点：
1. 照片文件是否在 `photos/` 文件夹内
2. `photo` 路径是否写对（区分大小写）
3. 是否已推送到线上

### Q：想改配色/样式？

A：编辑 `index.html` 中 `<style>` 标签里的 CSS。主体色是 `#0f0f1a`（深色背景）、`#ff6b6b`（高亮红色）。

### Q：占位图不好看？

A：打开 `generate-photos.html`，一键生成所有人的占位图并下载，放入 `photos/` 文件夹。

### Q：能绑定自己的域名吗？

A：GitHub Pages 和 Gitee Pages 都支持绑定自定义域名。在 Settings → Pages → Custom domain 填入即可。

## 九、技术要点

| 项 | 说明 |
|------|------|
| 布局 | 单列大图流，4:5 图片比例，类小红书风格 |
| 扫码定位 | URL 参数 `?person=N` + 平滑滚动 + 高亮动画 |
| 降级机制 | 照片加载失败自动切换到 Canvas 占位图 |
| 移动端 | iOS Safari / Android Chrome / 微信内置浏览器全适配 |
| 安全区域 | iPhone 刘海屏 + 底部横条自动避让 |
| 打印 | QR 码页内置打印样式，每码一页 |
| 缓存 | 内置 no-cache 声明，每次扫码都拉最新版本 |
